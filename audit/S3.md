## Dials (architect-locked + decisions.md)

### D1 — WEIGHTS == critical:50, high:15, medium:5, low:1 — PASS
core/scoring.py:17 `WEIGHTS = {"critical": 50, "high": 15, "medium": 5, "low": 1}`.
Live dump prints exactly this dict. ONE critical (50) > TWO highs (30):
test `test_one_critical_outranks_two_highs` (tests/test_scoring.py:44) and live
probe both confirm `50 > 30 == True`. Matches decisions.md S3 + prompts.md
Prompt #20 (architect dial fix #1).

### D2 — Severity floor; floor only worsens — PASS
Logic at core/scoring.py:52 `grade = _GRADES[max(_band_rank(value), _floor_rank(findings))]`.
`_floor_rank` (core/scoring.py:39): any critical ⇒ 3 (D); else any high ⇒ 2 (C);
else 0. `max(...)` means the floor can only RAISE the rank (worsen), never lower it.
Live: lone high → (15, **C**) not B; lone critical → (50, **D**); lone medium →
(5, **B**) (no floor). Worsen-only proven: 3 criticals band_rank=4(F) vs floor=3(D)
→ max → **F** (floor did not pull F down to D); 5 highs value=75 band=F vs floor=C
→ **F**. Test `test_severity_floor` (tests/test_scoring.py:61) asserts lone-high==C,
lone-critical==D, 3-critical==F.

### D3 — decisions.md S3 section AGREES with code — PASS
